The Spaire sandbox is a completely separate server from production. It has its own database, its own API, and its own accounts. Nothing you do in the sandbox touches your live data, customers, or balance.

Use it to build your integration before going live, test edge cases and failure scenarios, and run automated tests against a real API without worrying about side effects.

<Note>
  The sandbox is a separate environment, not a test mode toggle. As a Merchant of Record handling real financial transactions, Spaire keeps test data and live data on entirely separate infrastructure — this prevents any possibility of test records contaminating production accounting.
</Note>

## Access the sandbox

Go to [sandbox.spairehq.com](https://sandbox.spairehq.com) or click **Go to sandbox** from the organization switcher in your dashboard.

You'll need to create a separate account and organization in the sandbox. Your production credentials don't work there. This is intentional — and a feature: you can create as many sandbox accounts and organizations as you need, with no limits.

## Test payments

The sandbox runs on Stripe's test infrastructure. Use these card numbers to simulate different payment outcomes:

| Scenario | Card number |
|----------|-------------|
| Successful payment | `4242 4242 4242 4242` |
| Card declined | `4000 0000 0000 0002` |
| Insufficient funds | `4000 0000 0000 9995` |
| 3D Secure required | `4000 0025 0000 3155` |

For all test cards, use any future expiration date and any 3-digit CVC.

See Stripe's [full test card reference](https://docs.stripe.com/testing#cards) for more scenarios including international cards and specific decline codes.

## API and SDK

The sandbox API base URL is `https://sandbox-api.spairehq.com`.

Generate an access token from your **sandbox** organization settings — production tokens don't work in sandbox, and sandbox tokens don't work in production.

<CodeGroup>

```ts TypeScript
import { Spaire } from '@spaire/sdk'

const spaire = new Spaire({
  accessToken: process.env.SPAIRE_ACCESS_TOKEN,
  server: 'sandbox',
})
```

```python Python
from spaire_sdk import Spaire

s = Spaire(
    server="sandbox",
    access_token="<YOUR_BEARER_TOKEN_HERE>",
)
```

```go Go
s := spairego.New(
    spairego.WithServer(spairego.ServerSandbox),
    spairego.WithSecurity(os.Getenv("SPAIRE_ACCESS_TOKEN")),
)
```

```php PHP
$sdk = Spaire::builder()
    ->setServer('sandbox')
    ->setSecurity(getenv('SPAIRE_ACCESS_TOKEN'))
    ->build();
```

```ruby Ruby
s = OpenApiSDK::Spaire.new(
  server_url: 'https://sandbox-api.spairehq.com',
  access_token: ENV['SPAIRE_ACCESS_TOKEN']
)
```

</CodeGroup>

## Limitations

These only apply to sandbox. Production behavior is not affected.

- Subscriptions are automatically canceled **90 days after creation**
- Payout operations are simulated — no real funds are transferred
- Webhook delivery may have higher latency than production
