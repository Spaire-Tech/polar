In Spaire, everything you sell is a **product**. Whether it's a one-time download, a monthly SaaS plan, or a usage-metered API ‚Äî the product is the core object that holds your pricing, benefits, and billing configuration.

## Create a product

Navigate to **Catalog ‚Üí Products** and click **New product**.

### Name and description

- **Name** ‚Äî shown to customers at checkout and on their receipts
- **Description** ‚Äî optional, Markdown supported, shown at checkout

### Pricing

Every product has a **billing cycle** and a **price type**. These are locked after creation ‚Äî if you need different terms, create a new product.

**Billing cycles:**

| Cycle | When to use |
|-------|-------------|
| One-time | Single charge, no renewal |
| Monthly | Recurring charge every month |
| Yearly | Recurring charge every year |

**Price types:**

| Type | Description |
|------|-------------|
| `fixed` | A set amount ‚Äî e.g., $29/month |
| `free` | No charge ‚Äî free tiers, trials, or open-access products |
| `custom` | Pay-what-you-want with optional min/max bounds |
| `seat_based` | Billed per seat, tiered by seat count |
| `metered_unit` | Billed per unit consumed, measured by a meter |

Spaire supports 36 currencies. Set your price in any supported currency ‚Äî AED, ARS, AUD, BRL, CAD, CHF, CLP, CNY, CZK, DKK, EUR, GBP, HKD, HUF, ILS, INR, JPY, KRW, MXN, MYR, NOK, NZD, PLN, RON, SEK, SGD, THB, TWD, UAH, USD, and more.

<Info>
  Billing cycle and price type cannot be changed after a product is created. Create a new product if you need different terms. You can always present multiple products ‚Äî e.g., monthly and yearly ‚Äî together at checkout.
</Info>

### Trial period

For recurring products, you can offer a free trial before the first charge. Enable **Trial period** and set the duration in days, weeks, months, or years.

During a trial, the subscription status is `trialing`. Billing begins automatically when the trial ends. See [Trials](/features/trials) for more.

### Product media

Upload images to display on your product page and at checkout. Each image can be up to 10 MB. You can reorder and remove images at any time.

### Checkout fields

Collect additional information from customers at checkout ‚Äî phone numbers, company names, terms acceptance, or anything else you need. Fields are created at the organization level in **Settings ‚Üí Custom Fields** and then enabled per product.

See [Custom Fields](/features/custom-fields) for the full setup guide.

### Benefits

Attach automated entitlements to your product. When a customer purchases, Spaire immediately grants access ‚Äî no fulfillment code required for the built-in types.

Available benefit types:

| Type | What Spaire does |
|------|-----------------|
| `license_keys` | Generates a unique license key and delivers it |
| `meter_credit` | Adds credits to the customer's usage meter balance |
| `discord` | Assigns a Discord role via OAuth |
| `downloadables` | Grants access to uploaded files |
| `github_repository` | Grants access to a GitHub repository |
| `custom` | Calls a webhook URL you define to provision and revoke |

See [Benefits](/features/benefits/introduction) for setup details on each type.

## Multiple prices for one product

Spaire doesn't have "variants" in the traditional sense. Instead, create separate products for each pricing model ‚Äî monthly and yearly, for example ‚Äî and present them together at checkout.

Both [checkout links](/features/checkout/links) and [checkout sessions](/features/checkout/session) support multiple products, letting customers switch between them before paying.

## Update a product

You can edit name, description, media, benefits, and custom fields at any time.

**Price changes:** For fixed-price products, you can change the amount. Existing subscribers stay on their current price ‚Äî the new price applies only to new subscribers.

**Adding benefits:** Existing subscribers get new benefits automatically.

**Removing benefits:** Existing subscribers lose access to removed benefits immediately.

## Archive a product

Products can't be deleted, but they can be **archived**. Archived products stop appearing for new purchases, but existing subscribers keep their access and subscriptions continue to renew.

To archive: open the product and click **Archive** at the bottom of the page.

To unarchive: use the [Products API](/api-reference/products/update) with `is_archived: false`.
